## 3. 사용자 워크플로우 (반드시 지킬 것)

### A. 파일 전달 방식
- 사용자는 **Claude가 만든 파일을 다운로드**해서 본인 로컬 ExEAT 폴더에 옮김
- 즉 너는 코드를 채팅창에 붙여넣지 말고, **반드시 `create_file`로 파일을 만들고 `present_files`로 다운로드 가능하게 제공**
- 한 번에 너무 많은 파일을 만들지 말고, **한 STEP 단위로 제공**

### B. 한 STEP씩 진행 + 사용자 확인 후 다음 STEP
- 절대 여러 STEP을 한꺼번에 진행하지 않는다
- 한 STEP 끝나면:
  1. `docs/PROGRESS.md` 와 `docs/ARCHITECTURE.md` 를 업데이트한 새 버전을 함께 제공
  2. 사용자에게 **로컬에서 어떻게 실행/확인**하는지 안내
  3. 사용자 확인 후 다음 STEP 진행

### C. 환경변수
- API 키는 `.env` 파일에 두며 git에 커밋되지 않는다 (`.gitignore` 처리)
- 사용자에게는 `.env.example` 만 제공
- API 키를 채팅창이나 코드 안에 절대 하드코딩하지 않는다

### D. 톤
- 사용자(일하)는 친근한 반말로 대화하길 원함
- 직설적, 솔직한 피드백 선호 — "이거 안 좋아 보이는데" 솔직히 말해도 됨
- 과한 설명보다 핵심 + 행동 지시 + 다음 단계 확인질문 구조 선호

## 5. 합의된 기능 7개 (F1, F2, F3, F4, F6, F7, F9)

> F5, F8 은 **명시적으로 제외**됨. 다시 추가하지 말 것.

| ID | 이름 | 한 줄 설명 |
|---|---|---|
| F1 | EXIT 타이밍 + 악성 재고 방지 | 수명주기 단계 + 발주 안전선 |
| F2 | 트렌드 재료 진입 판단 | GO/WAIT/STOP |
| F3 | 트렌드 그래프 + 평균 패턴 비교 | 스테디형 vs 급락형 매칭 |
| F4 | 지역 적합도 분석 | 거주 vs 소비 연령 매칭 |
| F6 | 손익분기 시뮬레이터 | 손익분기 vs EXIT 시점 비교 |
| F7 | 과거 트렌드 케이스 라이브러리 | 대만카스테라 등 사례 |
| F9 | 자유 질문 박스 | 통합 분석 메인 진입점 |

## 6. 디자인 방향 (잠정)

- 톤: **화이트 미니멀, 진단서 느낌** (B안)
- chadcn/ui 라이브러리는 **사용 안 함**, 다만 "기억에 남는 인터랙션" 정신은 일부 차용
- 모바일 반응형 필수 (~480px / 481~768px / 769px+ 3 단계)

## 7. 첫 인수 인사 템플릿

너가 처음 인수하면 사용자에게 이렇게 인사해라:

> "이전 Claude가 작업한 ExEAT 프로젝트 인수했어. PROGRESS.md 보니까 [현재 STEP]까지 끝났고, 다음은 [다음 STEP] 차례야. [이 STEP 에서 할 작업 요약] 진행할게. 시작 전에 확인할 거 [있으면 질문 / 없으면 시작 알림]."

이렇게 해야 사용자가 "이 Claude가 맥락 파악했구나" 확인하고 안심할 수 있음.

## 8. 사용자가 자주 하는 질문 / 패턴

- "이거 좀 별로 아니야?" — 진심으로 의심하는 게 아니라 검증 중. 솔직한 답 원함
- "다른 방법 있어?" — 현재 방향 재고하자는 신호
- "흔한 거 아니야?" — 차별점을 명확히 언어화해달라는 요청
- "너로 다 개발할거야" — 코드 직접 짜라는 뜻 (사용자는 수동 코딩 안 함)
