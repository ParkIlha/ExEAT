## 📡 데이터 소스 보강 (현재 검색량 단일 의존)

### 🟡 네이버 쇼핑인사이트 API
- **왜**: 검색은 관심도, 쇼핑 클릭은 **구매 직전 시그널**. 재료/디저트 카테고리 추적 시 진짜 수요 곡선 확보
- **어떻게**: `https://openapi.naver.com/v1/datalab/shopping/category/keywords` — 이미 발급된 NAVER 키 그대로
- **난이도**: ★ (30분)
- **임팩트**: 검색량과 다른 곡선 나오면 차별화 포인트 큼

### 🟡 공공데이터 - 소상공인 상권정보
- **왜**: "트렌드 따라 창업한 카페가 진짜 폐업하는지" 데이터로 증명 가능. 대만카스테라 사례 정량화
- **어떻게**: data.go.kr `소상공인시장진흥공단_상가(상권)정보` 또는 `상권분석`
- **난이도**: ★★ (인증키 발급 + 데이터 가공 1~2시간)
- **임팩트**: 해커톤 차별화 무기. "검색량 하락 + 폐업률 상승" 동시 시각화 가능

### 🟡 행정안전부 주민등록 인구통계 (F4 계획됨)
- **왜**: 지역 적합도 분석. "이 지역은 20대가 70% → 트렌드 메뉴 적합" 같은 판정
- **어떻게**: jumin.mois.go.kr 월별 CSV 다운로드 → `backend/data/population.csv`
- **난이도**: ★ (CSV 정적 파일)

### ⏸ 네이버 블로그/카페 검색 API
- **왜**: 후기 텍스트에서 negative 시그널 추출 ("별로", "다신 안 먹어")
- **어떻게**: `openapi.naver.com/v1/search/blog.json` + Claude 감정 분석
- **난이도**: ★★ (Claude 호출 1회 더)

### ⏸ 카드 결제 데이터 (신한/BC)
- **왜**: 가장 정확한 실 소비 데이터
- **어떻게**: 신한SOL Bank API 또는 공공데이터
- **난이도**: ★★★ (발급 까다로움, 해커톤 시간 부족)

### ❌ 인스타그램/유튜브 트렌드
- 비공식 API 또는 유료. 해커톤 범위 밖.

## 🎯 기능 아이디어

### ⏸ 트렌드 키워드 자동 추천
- **왜**: "뭘 검색해야 할지 모르겠다" 사용자 대응
- **어떻게**: 네이버 쇼핑인사이트 식품 카테고리 인기 검색어 Top 10 노출
- **난이도**: ★

### ⏸ 알림 (트렌드 EXIT 임박 시)
- **왜**: "내가 도입한 메뉴의 EXIT 시점 알려주기"
- **어떻게**: 즐겨찾기 기능 + 이메일/푸시 (해커톤은 어려움, 데모만)
- **난이도**: ★★★

### ⏸ 비교 모드 (키워드 2~3개 동시 분석)
- **왜**: "두바이초콜릿 vs 크로플 vs 탕후루 누가 먼저 EXIT?"
- **어떻게**: DataLab `keywordGroups`에 여러 개 넣어서 한 그래프에 겹치기
- **난이도**: ★★
